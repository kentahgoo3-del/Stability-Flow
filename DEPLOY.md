# StabilityFlow — Deployment Guide
## GitHub → Render → Cloudflare

---

## PART 1 — Push to GitHub (5 minutes)

### Step 1: Create the GitHub repository
1. Go to **github.com** and sign in
2. Click **+** (top right) → **New repository**
3. Name: `stabilityflow`
4. Set to **Private**
5. Leave everything else blank — **do NOT** tick "Add README" or "Add .gitignore"
6. Click **Create repository**
7. Copy the repo URL shown (looks like `https://github.com/YOUR-USERNAME/stabilityflow.git`)

### Step 2: Connect Replit to GitHub
1. In Replit, click the **Git** icon in the left sidebar
2. Click **Connect to GitHub** and authorize Replit
3. Select the `stabilityflow` repo you just created
4. Click **Push to GitHub**

✅ Your code is now on GitHub.

---

## PART 2 — Deploy on Render (15 minutes)

### Step 3: Create the Web Service on Render
1. Go to **dashboard.render.com** and sign in
2. Click **New +** → **Blueprint**
3. Connect your GitHub account if not already connected
4. Select the `stabilityflow` repository
5. Render will detect the `render.yaml` file automatically
6. Click **Apply** — this creates BOTH the database AND the web service in one go

> The `render.yaml` file already prepared in your repo configures everything:
> build command, start command, environment variables, and the database.

### Step 4: Wait for the first deploy
- Render will install packages, build the app, and start it (~5–10 minutes)
- You can watch the build log in real time
- When it says **"Live"** with a green dot, it's ready
- Your app URL will be: `https://stabilityflow.onrender.com` (or similar)

### Step 5: Verify it works
- Open the Render URL in your browser
- You should see the StabilityFlow login screen
- The database tables are created automatically on first boot

✅ App is live on Render.

---

## PART 3 — Custom Domain via Cloudflare (10 minutes)

### Step 6: Buy or confirm your domain in Cloudflare
- If you don't have a domain yet: **dash.cloudflare.com** → **Domain Registration** → search and purchase
- If you already own one: make sure it's added to Cloudflare (nameservers pointing to Cloudflare)

### Step 7: Add the custom domain in Render
1. In Render → your `stabilityflow` service → **Settings** tab
2. Scroll to **Custom Domains** → click **Add Custom Domain**
3. Type your domain, e.g. `app.yourdomain.com` (using a subdomain is easiest)
   - For root domain (`yourdomain.com`) use `@` as the Cloudflare DNS name
4. Render shows you a **CNAME value** — copy it (looks like `stabilityflow.onrender.com`)

### Step 8: Add DNS record in Cloudflare
1. Cloudflare dashboard → your domain → **DNS** → **Records** → **Add record**
2. Fill in:
   - **Type**: `CNAME`
   - **Name**: `app` (or `@` for root domain)
   - **Target**: paste the value from Render (e.g. `stabilityflow.onrender.com`)
   - **Proxy status**: **Proxied** (orange cloud ON) ← important for SSL
3. Click **Save**

### Step 9: Confirm in Render
1. Back in Render → Custom Domains
2. Wait ~2–5 minutes, then click **Verify**
3. It should turn green — SSL is handled automatically by Cloudflare

✅ Your app is live at `https://app.yourdomain.com`

## PART 4 — Keeping it updated

Every time you push code changes from Replit to GitHub, **Render auto-deploys** — no manual steps needed.

Workflow:
```
Edit in Replit → Push to GitHub (Git sidebar) → Render redeploys automatically
```

## Troubleshooting

**Build fails on Render:**
- Check the build log for errors
- Most common cause: a missing package — add it and push again

**App loads but database is empty:**
- The app auto-runs migrations and seed on first boot
- If it didn't, go to Render → your service → **Shell** → run `npm run db:push`

**Domain shows "not secure" or SSL error:**
- Make sure the Cloudflare proxy is **ON** (orange cloud)
- Wait 10 minutes for DNS to propagate

**App goes to sleep (free plan):**
- Render free tier sleeps after 15 minutes of inactivity
- Upgrade to **Starter plan** ($7/mo) for always-on
